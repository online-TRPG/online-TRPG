# TRPG 온라인 플랫폼 노드별 UI 구성 정리

## 0. 전체 전제

플랫폼의 시나리오는 여러 개의 노드로 구성된다.

각 노드는 하나의 장면에 해당하며, 타입은 다음 3가지로 나뉜다.

```text
스토리 노드
탐색 노드
전투 노드
```

전체 화면의 기본 구조는 다음과 같다.

```text
┌──────────────────────────────┬──────────────┐
│                              │              │
│        메인 화면 영역          │   우측 채팅창  │
│  장면 이미지 / 탐색맵 / 전투맵  │   로그 / 대화  │
│                              │              │
├──────────────────────────────|              |
│          하단 UI 영역         |              │
└─────────────────────────────────────────────┘
```

하단 UI는 노드 타입에 따라 다르게 구성한다.

---

# 1. 스토리 노드 UI

## 1-1. 스토리 노드의 목적

스토리 노드는 전투나 맵 탐색보다 **장면 연출과 대화 몰입감**이 중요하다.

GM이 현재 노드에 세팅한 요소를 메인 화면에 보여준다.

예시:

- 배경 이미지
- NPC 이미지
- 아이템 이미지
- 사건 장면 이미지
- 분위기용 오브젝트 이미지

## 1-2. 하단 UI 핵심 방향

스토리 노드 하단은 조작 버튼을 많이 넣는 구조가 아니라, **파티 캐릭터들이 장면 안에 존재하는 느낌**을 주는 방향이 적절하다.

추천 구조:

```text
┌──────────────────────────────────────────────────┐
│  캐릭터 카드  캐릭터 카드  캐릭터 카드  캐릭터 카드  │
│     말풍선       말풍선       말풍선       말풍선  │
└──────────────────────────────────────────────────┘
```

## 1-3. 캐릭터 카드 + 말풍선 구조

스토리 노드에서는 파티 캐릭터 카드를 하단에 배치하고, 플레이어가 채팅을 입력하면 해당 캐릭터 카드 위에 말풍선이 뜨는 방식이 적절하다.

다만 우측 채팅창과 역할을 명확히 분리해야 한다.

### 우측 채팅창 역할

```text
기록 / 로그 / 전체 대화 확인
```

- 모든 대화가 시간순으로 저장됨
- 시스템 메시지와 GM 설명까지 포함
- 스크롤해서 이전 대화 확인 가능
- 전투/판정 로그도 확인 가능

### 캐릭터 말풍선 역할

```text
현재 장면의 연출 / 누가 말하는지 시각화
```

- 각 캐릭터 최근 발화 1개만 표시
- 캐릭터가 실제로 말하는 느낌 제공
- 장면 몰입감 강화
- 일정 시간 후 사라짐
- 전체 로그 역할은 하지 않음

즉, 채팅창은 **기록**, 말풍선은 **연출**이다.

## 1-4. 말풍선 사용 규칙

말풍선은 다음 방식으로 제한하는 것이 좋다.

- 캐릭터 발화만 표시
- 캐릭터 최근 대사 1개만 표시
- 대사 길이에 따라 길어지나 최대 길이 제한
- 클릭 시 우측 채팅창의 탭이 메인 탭으로 변경됨
- 5~8초 후 자동 페이드아웃
- 채팅 탭에서 작성하는 OOC 대화는 말풍선으로 표시하지 않음
- 시스템 메시지, 판정 결과, 전투 로그는 말풍선으로 표시하지 않음

## 1-5. 캐릭터 카드에 들어갈 정보

스토리 노드의 캐릭터 카드는 간단해야 한다.

추천 정보:

```text
캐릭터 초상
캐릭터명
플레이어명
간단 상태 아이콘
```

예시:

```text
[카엘]
출혈 / 중독
"이 배신자 녀석... 뒤통수를 치다니!!"
```

### 현재 발화자 표시

발화 시 해당 캐릭터 카드 테두리를 강조한다.

### 장면 참여 NPC 카드

중요 NPC가 있는 장면이라면 하단이나 메인 화면 근처에 NPC 카드를 작게 표시할 수 있다.

```text
[NPC: 마을 촌장]
"우물 근처에는 가지 마시오."
```

NPC가 많아지면 복잡해지므로 주요 NPC 1~3명만 표시하는 것이 좋다.

## 1-7. 스토리 노드에서 빼는 것이 좋은 요소

- 전체 인벤토리
- 캐릭터 상세 능력치
- 맵 조작 버튼
- 전투 행동 버튼
- 긴 로그
- 전체 단서 목록
- 전체 NPC 목록
- 복잡한 판정 패널

이런 요소는 별도 창, 캐릭터 카드 클릭, 우측 패널 등으로 처리한다.

## 1-8. 스토리 노드 최종 구조

```text
┌──────────────────────────────┬──────────────┐
│                              │              │
│        메인 장면 이미지       │   채팅/로그   │
│   NPC / 배경 / 아이템 이미지  │               │
│                              │              │
├──────────────────────────────|              |
│  파티 캐릭터 카드 영역        |               │
│  - 최근 대사 말풍선           |               │
│  - 현재 발화자 강조           |               │
│  - 간단 상태 아이콘           |               │
│  - 공개 단서 알림             |               │
└─────────────────────────────────────────────┘
```

# 2. 탐색 노드 UI

## 2-1. 탐색 노드의 목적

탐색 노드는 플레이어가 맵 위에서 자기 캐릭터를 움직이며 장소, 물체, NPC, 단서를 조사하는 노드다.

핵심은 다음이다.

```text
어디로 이동할 것인가
무엇을 조사할 것인가
무엇과 상호작용할 것인가
무엇을 획득하거나 사용할 것인가
```

## 2-2. 메인 화면 구성

탐색 노드의 메인 화면은 탐색맵이 차지한다.

캐릭터 카드는 메인 화면의 맵 좌측 중앙에 작은 카드로 간단히 표시한다.

예시:

```text
┌──────────────────────────────┬──────────────┐
│ [파티 카드]                   │              │
│ [작은 카드]     탐색맵          │   채팅/로그   │
│ [작은 카드]                   │              │
└──────────────────────────────┴──────────────┘
```

## 2-3. 하단 UI 핵심 방향

탐색 노드 하단은 다음 3분할 구조가 적절하다.

```text
┌────────────────────────────────────────────────────┐
│ 선택 대상: 낡은 나무문 / 잠겨 있음 / 수상한 흠집 있음 │
├──────────────┬──────────────────────┬──────────────┤
│ 내 상태        │ 상황별 행동 버튼        │ 빠른 인벤토리   │
│ HP 18/24      │ 열기  조사  해제  부수기 │ 포션 열쇠 횃불  │
│ 정상 / 비은신  │ 핑  거리재기  요청       │ 전체보기        │
└──────────────┴──────────────────────┴──────────────┘
```

비율은 다음 정도가 적절하다.

```text
내 상태 20%
행동 버튼 55%
인벤토리 25%
```

## 2-4. 좌측: 내 상태

탐색 노드에서 하단 좌측에는 내 캐릭터 상태를 둔다.

추천 정보:

```text
HP 18/24
상태: 정상
은신 아님
```

압축형:

```text
HP 18/24 | 정상 | 은신 아님
```

추가 가능 정보:

- 주요 상태이상
- 은신 여부
- 집중 여부
- 빛/어둠 영향
- 위험 상태

## 2-5. 중앙: 상황별 행동 버튼

탐색 노드의 핵심은 중앙 행동 버튼이다.

기본 버튼 예시:

```text
이동
조사
상호작용
줍기
사용
대화
핑
거리 재기
```

중요한 점은 모든 버튼을 항상 보여주는 것이 아니라, **선택 대상에 따라 버튼을 바꾸는 것**이다.

### 빈 땅 선택 시

```text
[이동] [핑 찍기]
```

### 문 선택 시

```text
[이동] [핑 찍기] [열기] [조사] [잠금 해제] [부수기]
```

### 상자 선택 시

```text
[이동] [핑 찍기] [열기] [조사] [잠금 해제] [부수기] [들기]
```

### NPC 선택 시

```text
[이동] [핑 찍기] [대화] [관찰]
```

### 아이템 선택 시

```text
[이동] [핑 찍기] [줍기] [조사]
```

### 수상한 지형 선택 시

```text
[이동] [핑 찍기] [조사]
```

## 2-6. 우측: 빠른 인벤토리

인벤토리는 우측에 배치하는 것이 적절하다.

하단에는 **빠른 인벤토리** 정도만 둔다.

추천 예시:

```text
[인벤토리] 전체보기
포션 | 횃불 | 열쇠 |
밧줄 | 락픽 | 골드 |
```

이유:

- 전체 인벤토리를 하단에 펼치면 화면이 복잡해짐
- 맵 조작 공간이 줄어듦
- 행동 버튼이 밀림
- 아이템이 많아지면 가독성이 떨어짐

전체 인벤토리는 별도 창으로 연다.

## 2-7. 선택 대상 정보

탐색 노드에서는 “내가 지금 무엇을 선택했는가”가 중요하다.

하단 최상단에 얇은 정보 줄을 두는 것이 좋다.

예시:

```text
선택 대상: 낡은 나무문 / 상태: 잠겨 있음 / 수상한 흠집 있음
```

선택 대상 정보는 중앙 행동 버튼과 연결된다.

예:

```text
문 선택 → [열기] [잠금 확인] [함정 확인] [부수기]
열쇠 선택 → [문에 사용] [조사] [버리기]
```

## 2-8. 맵 보조 기능

탐색 노드에서 필요한 맵 보조 기능은 다음이다.

```text
핑
거리 재기
내 캐릭터 위치로 카메라 이동
파티 집결 요청
시야 초기화
```

MVP에서는 다음만 있어도 충분하다.

```text
핑
거리 재기
내 캐릭터로 이동
```

## 2-9. 탐색 노드에서 빼도 되는 요소

- 전투용 대형 스킬바
- 전체 인벤토리 상시 표시
- 복잡한 파티 진형 설정
- 자동 탐색
- 세부 행동 예약
- 아이템 제작
- 상세 장비 교체

## 2-10. 탐색 노드 최종 구조

```text
┌─────────────────────────────────────────────────────────┬───────────────┐
│ [파티 카드]                                              │               │
│ [작은 카드]       탐색맵                                 │   채팅/로그    │
│ [작은 카드]                                              │               │
├─────────────────────────────────────────────────────────┤               │
│ 선택 대상: 낡은 상자 / 상태: 잠겨 있음                     │               │
├──────────────┬──────────────────────┬───────────────────┤               │
│ 내 상태        │ 상황별 행동 버튼        │ 빠른 인벤토리   │               │
│ HP / 상태      │ 조사 열기 사용 줍기     │ 포션 열쇠 횃불  │               │
└──────────────┴──────────────────────┴───────────────────┴───────────────┘
```

# 3. 전투 노드 UI

## 3-1. 전투 노드의 목적

전투 노드는 전투맵 위에서 캐릭터, 적 몬스터, 지형지물, 범위 효과 등을 배치하고 전투를 진행하는 노드다.

탐색 노드와 유사한 하단 구조를 유지하되, 중앙 행동 버튼의 종류가 더 많아진다.

다만 전투 노드는 버튼만 늘리는 것으로는 부족하다.

전투에서 중요한 정보는 다음이다.

```text
누구 차례인가
내가 이번 턴에 무엇을 쓸 수 있는가
대상이 누구인가
대상에게 행동이 가능한가
사거리와 시야가 맞는가
행동 자원을 이미 사용했는가
턴을 끝냈는가
```

## 3-2. 전투 노드에서 추가로 필요한 필수 UI

탐색 노드에 비해 전투 노드는 다음 요소가 추가로 필요하다.

```text
턴 순서
현재 턴 표시
행동 자원 표시
선택 대상 전투 정보
사거리 / 시야 / 엄폐 정보
행동 확정 / 취소
턴 종료 버튼
전투 로그
반응 / 기회공격 알림
맵 위 이동 / 공격 범위 표시
```

## 3-3. 턴 순서 표시

전투에서는 턴 순서 표시가 필수다. 맵 상단에 턴 순서를 표시한다.

예시:

```text
현재 턴: 엘런
다음: 고블린 A → 리아 → 고블린 B → 카엘
```

UI 예시:

```text
[엘런] [고블린 A] [리아] [고블린 B] [카엘]
```

현재 턴 대상은 강조 표시한다.

## 3-4. 좌측: 내 전투 상태

전투 노드 하단 좌측은 탐색 노드의 “내 상태”보다 더 상세해야 한다.

필수 정보:

```text
HP
AC
이동력
행동 가능 여부
보너스 행동 가능 여부
반응 가능 여부
상태이상
```

예시:

```text
HP 18/24
AC 15
이동 6칸 남음
행동 가능
보너스 행동 가능
반응 가능
상태: 정상
```

압축형:

```text
HP 18/24 | AC 15 | 이동 6칸 | 행동 O | 보너스 O | 반응 O
```

## 3-5. 중앙: 전투 행동 버튼

전투 노드는 버튼이 많아지므로, 중앙 버튼 영역을 카테고리형으로 나누는 것이 좋다.

추천 카테고리:

```text
[공격] [이동] [기술/주문] [아이템] [기타]
```

### 공격 탭

```text
기본 공격
근접 공격
원거리 공격
비무장 공격
기회공격
```

### 이동 탭

```text
이동
질주
이탈
엎드리기
일어서기
```

### 기술/주문 탭

```text
스킬 사용
주문 사용
범위 지정
집중 유지/해제
```

### 아이템 탭

```text
포션 사용
아이템 던지기
무기 교체
도구 사용
```

### 방어/기타 탭

```text
방어
숨기
도움
대기
턴 종료
```

## 3-6. 우측: 빠른 인벤토리

전투 노드에서도 우측 인벤토리는 유지할 수 있다.

다만 탐색 노드보다 더 전투용으로 제한하는 것이 좋다.

예시:

```text
포션
투척 무기
스크롤
장착 무기
전체보기
```

전투 중 전체 인벤토리를 계속 펼쳐두면 판단할 정보가 너무 많아진다.

따라서 하단 우측에는 전투 중 즉시 쓸 수 있는 아이템만 둔다.

## 3-7. 선택 대상 전투 정보

전투에서는 선택 대상 정보가 탐색보다 중요하다.

하단 최상단 정보 줄에 표시한다.

예시:

```text
대상: 고블린 A | 거리 5칸 | 시야 확보 | 절반 엄폐 | HP: 부상
```

대상 HP 표시는 룰/GM 설정에 따라 다르게 할 수 있다.

### 추상형

```text
정상 / 부상 / 빈사
```

### 수치형

```text
HP 7/15
```

MVP에서는 추상형이 더 안전하다.

## 3-8. 사거리 / 시야 / 엄폐 정보

전투에서는 행동을 누르기 전에 가능한 행동인지 알 수 있어야 한다.

예시:

```text
장궁 공격
대상: 고블린 A
거리: 18m
사거리 안
시야 확보
명중 보정: +5
```

불가능한 경우:

```text
사거리 밖
시야 없음
대상이 엄폐 중
행동 자원 부족
```

이 정보는 버튼 비활성화와 함께 표시하는 것이 좋다.

## 3-9. 행동 확정 / 취소

전투에서는 실수 클릭이 치명적이다.

따라서 주요 행동에는 확정 단계가 필요하다.

예시:

```text
장궁 공격 선택
→ 대상 선택
→ 예상 정보 표시
→ [확정] [취소]
```

확정이 필요한 행동:

```text
공격
주문/스킬
아이템 사용
이동
범위 공격
턴 종료
```

## 3-10. 턴 종료 버튼

전투 노드에서는 턴 종료 버튼을 분명하게 보여줘야 한다.

```text
[턴 종료]
```

추천 위치:

```text
또는 하단 우측 인벤토리 옆
```

턴 종료는 매 턴 사용하는 버튼이므로 너무 작게 두면 안 된다.

## 3-11. 전투 로그

우측 채팅창은 메인 탭은 전투 로그를 함께 담당할 수 있다.

전투 로그 메시지에 별도 색상/아이콘을 적용한다.

전투 로그 예시:

```text
엘런이 고블린 A를 공격
명중 굴림: 17 + 5 = 22
명중
피해: 8
고블린 A 부상
```

## 3-12. 반응 / 기회공격 알림

D&D 계열 룰을 참고한다면 반응 처리가 중요하다.

예시:

```text
고블린 A가 이동합니다.
기회공격 가능
[기회공격] [넘기기]
```

이런 선택지는 하단 고정 버튼보다는 팝업성 UI가 적절하다.

추천 위치:

```text
맵 중앙 하단 작은 확인창
```

## 3-13. 전투 노드에서 필요한 맵 위 시각 피드백

전투는 하단 버튼만으로 처리하면 안 된다.

맵 위에 다음 정보가 함께 표시되어야 한다.

```text
이동 가능 범위
공격 사거리
범위 공격 템플릿
시야 표시
엄폐 표시
위험 지역 표시
선택 대상 테두리 강조
현재 턴 캐릭터 강조
```

## 3-14. 전투 노드 최종 구조

```text
┌───────────────────────────────────────────────────────────┬──────────────┐
│ 턴 순서: 엘런 → 고블린A → 리아                              │              │
│                                                           │   채팅/로그   │
│            전투맵                                          │ [채팅/전투로그]│
│                                                            │              │
├────────────────────────────────────────────────────────────┤              │
│ 대상: 고블린 A | 거리 5칸 | 시야 확보 | 절반 엄폐             │              │
├──────────────┬────────────────────────────┬────────────────┤              │
│ 내 전투 상태   │ 전투 행동 버튼             │ 빠른 인벤토리   │              │
│ HP / AC       │ 공격 이동 스킬 주문 기타    │ 포션 투척물 등  │              │
│ 행동자원       │ 대기 도움 숨기 턴종료       │ 전체보기       │              │
└──────────────┴─────────────────────────────┴───────────────┴──────────────┘
```

비율:

```text
내 상태 22%
행동 버튼 58%
인벤토리 20%
```

# 4. 노드별 하단 UI 비교

## 4-1. 스토리 노드

```text
목적: 장면 연출과 대화 몰입
하단: 파티 캐릭터 카드 + 말풍선
핵심: 채팅창은 기록, 말풍선은 연출
```

필수 요소:

- 캐릭터 카드
- 최근 발화 말풍선
- 현재 발화자 강조
- 간단 상태 아이콘

## 4-2. 탐색 노드

```text
목적: 맵 탐색과 오브젝트 상호작용
하단: 좌측 내 상태 / 중앙 상황별 행동 / 우측 빠른 인벤토리
핵심: 선택 대상에 따라 행동 버튼 변경
```

필수 요소:

- 내 상태
- 선택 대상 정보
- 상황별 행동 버튼
- 빠른 인벤토리
- 핑
- 거리 재기

## 4-3. 전투 노드

```text
목적: 턴 기반 전투 진행
하단: 좌측 내 전투 상태 / 중앙 전투 행동 / 우측 전투용 빠른 인벤토리
핵심: 행동 자원, 턴 순서, 대상 정보, 사거리/시야 판단
```

필수 요소:

- 현재 턴 강조
- 내 전투 상태
- 행동 자원
- 선택 대상 전투 정보
- 사거리/시야/엄폐 정보
- 전투 행동 버튼
- 턴 종료
- 전투 로그
- 반응/기회공격 알림
- 맵 위 범위 표시

# 5. 최종 권장 방향

## 5-1. 공통 원칙

노드별 하단 UI는 완전히 다르게 만들기보다, 기본 구조를 유지하면서 목적에 맞게 변형하는 것이 좋다.

```text
스토리: 연출 중심
탐색: 상호작용 중심
전투: 행동 자원과 대상 판단 중심
```

## 5-2. 추천 구조 요약

```text
스토리 노드
→ 캐릭터 카드 + 말풍선

탐색 노드
→ 내 상태 / 상황별 행동 / 빠른 인벤토리

전투 노드
→ 내 전투 상태 / 카테고리형 전투 행동 / 전투용 빠른 인벤토리
```

## 5-3. 가장 중요한 기준

### 스토리 노드

```text
대화가 누구의 말인지 직관적으로 보여줄 것
```

### 탐색 노드

```text
선택한 대상에 대해 지금 무엇을 할 수 있는지 보여줄 것
```

### 전투 노드

```text
내 턴에 무엇을 쓸 수 있고, 대상에게 가능한 행동인지 즉시 알 수 있게 할 것
```
