# TRPG 온라인 플랫폼 전투 UI 구현 순서 정리

## 0. 최종 추천 UI 구성

```text
┌──────────────────────────────────────────────┬────────────────────┐
│ 턴 순서 바                                    │ 우측 채팅창         │
│ [라운드] [현재 턴] [다음 턴 목록]              │ [메인] [일반] [설정] │
├──────────────────────────────────────────────┤                    │
│                                              │                    │
│  좌측 파티 초상화                             │                    │
│  ┌─────────┐                                  │                    │
│  │ 캐릭터1 │                                  │                    │
│  ├─────────┤                                  │                    │
│  │ 캐릭터2 │          전투맵                   │                    │
│  ├─────────┤     토큰 / 적 / 지형지물          │                    │
│  │ 캐릭터3 │                                  │                    │
```

### UI 책임 분리

```text
대화: 우측 채팅창
전투 상황 확인: 상단 턴 순서 바 / 좌측 파티 초상화 / 대상 정보 패널
행동 선택: 하단 행동 탭
대상 선택: 전투맵과 토큰
상태 변경: 백엔드 검증
결과 묘사: GM 또는 AI
```

## 1. MVP 전투 UI 최종 기준

### 화면 구성

```text
좌측: 파티 초상화
상단: 턴 순서 바
중앙: 전투맵 / 토큰 / 지형지물
우측: 메인 채팅 / 일반 채팅
하단: [일반] [능력] [아이템] 행동 탭
하단 근처: 행동 자원 표시
맵 또는 하단 중앙: 대상 선택 안내
토큰 클릭 시: 대상 정보 패널
행동 버튼 hover/click 시: 행동 상세 설명
우측 또는 하단 구석: 설정 톱니바퀴
```

### MVP 포함 기능

- 전투맵 표시
- 토큰 표시
- 지형지물 표시
- 좌측 파티 초상화
- 턴 순서 바
- 행동 자원 표시
- 하단 행동 탭
- 행동 상세 설명
- 행동 불가 사유 표시
- 대상 선택 안내
- 대상 정보 패널
- 행동 확정 / 취소
- 턴 종료 버튼
- 기본 맵 조작
  - 드래그 이동
  - 휠 줌
  - 토큰 클릭
  - 빈 칸 클릭
  - 선택 해제
- 현재 조작 가능한 캐릭터 표시

### MVP 제외 기능

- 전투 로그
- 고급 지형 효과
- 시야 / 안개
- 복잡한 반응 행동
- 단축키 커스터마이징
- 고급 애니메이션
- 상세 피해 예측
- 리플레이 기능

## 3. 1단계: 전투 화면 레이아웃 골격

### 목표

전투 화면의 큰 구역을 먼저 고정한다.

### 구현할 영역

```text
CombatPage
├─ TurnOrderBar
├─ LeftPartyPortraits
├─ CombatMapArea
├─ RightChatSidebar
├─ BottomActionPanel
└─ SettingsButton
```

### 우선 구현 내용

- 전투 화면 전체 wrapper
- 좌측 파티 초상화 영역
- 상단 턴 순서 영역
- 중앙 맵 영역
- 우측 채팅 영역
- 하단 행동 UI 영역
- 설정 톱니바퀴 버튼

### 이 단계에서 하지 말 것

- 실제 전투 로직
- 공격 처리
- 이동 처리
- 스킬 처리
- 판정 계산

### 완료 기준

- 화면 비율이 고정된다.
- 전투맵 영역과 채팅창이 서로 침범하지 않는다.
- 하단 행동 패널이 맵을 과하게 가리지 않는다.

---

## 4. 2단계: 전투 상태 데이터 구조 정의

### 목표

프론트가 전투 화면을 그리는 데 필요한 데이터를 먼저 확정한다.

### CombatState 기본 구조 예시

```ts
type CombatState = {
  combatId: string;
  round: number;
  currentTurnActorId: string;
  turnOrder: TurnActor[];
  actors: CombatActor[];
  map: CombatMap;
  objects: MapObject[];
};
```

### CombatActor 예시

```ts
type CombatActor = {
  id: string;
  type: 'PLAYER' | 'NPC' | 'MONSTER';
  name: string;
  portraitUrl?: string;
  tokenUrl?: string;
  hp: number;
  maxHp: number;
  ac?: number;
  position: {
    x: number;
    y: number;
  };
  conditions: string[];
  isCurrentTurn: boolean;
  isControllableByMe: boolean;
};
```

### TurnActor 예시

```ts
type TurnActor = {
  actorId: string;
  name: string;
  portraitUrl?: string;
  type: 'PLAYER' | 'NPC' | 'MONSTER';
  isCurrent: boolean;
};
```

### CombatMap 예시

```ts
type CombatMap = {
  id: string;
  imageUrl: string;
  width: number;
  height: number;
  gridSize: number;
};
```

### 완료 기준

- 전투 화면에 필요한 데이터가 하나의 CombatState로 정리된다.
- 화면 컴포넌트가 임시 mock 데이터로 렌더링 가능하다.

---

## 5. 3단계: 전투 상태 조회 API 연결

### 목표

전투 화면 진입 시 현재 전투 상태를 불러온다.

### 추천 API

```http
GET /api/v1/sessions/{sessionId}/combat/state
```

### 응답에 포함할 것

- 현재 라운드
- 현재 턴 actor
- 턴 순서
- 모든 전투 actor 상태
- 맵 정보
- 토큰 위치
- 지형지물 정보

### 프론트 처리

```text
CombatPage 진입
→ combat state 요청
→ loading 표시
→ state 저장
→ 화면 렌더링
```

### 완료 기준

- 새로고침해도 현재 전투 화면이 복구된다.
- mock 데이터가 아니라 API 데이터로 화면이 그려진다.

---

## 6. 4단계: 전투맵 렌더링

### 목표

중앙에 전투맵을 표시한다.

### 구현 내용

- 맵 이미지 렌더링
- 맵 영역 크기 계산
- 기본 배율 적용
- 맵 좌표계 정의
- 그리드 표시 여부 옵션화

### 좌표 기준

```text
맵 원본 좌표: x, y
화면 표시 좌표: scale, offset 적용
토큰 위치: map coordinate 기준
```

### 완료 기준

- 맵이 중앙 영역에 안정적으로 표시된다.
- 화면 크기가 바뀌어도 맵이 깨지지 않는다.
- 토큰을 올릴 수 있는 좌표 기준이 정해진다.

---

## 7. 5단계: 토큰 렌더링

### 목표

맵 위에 아군, 적, NPC 토큰을 표시한다.

### 구현 내용

- actor.position 기준으로 토큰 표시
- actor.type에 따라 테두리 또는 아이콘 구분
- 현재 턴 actor 강조
- 내가 조작 가능한 actor 강조
- 사망 / 빈사 상태 표시

### 토큰 표시 정보

```text
토큰 이미지
이름 또는 약칭
HP 간이 바
상태이상 아이콘
현재 턴 표시
```

### 완료 기준

- 맵 위에 모든 actor가 위치대로 표시된다.
- 현재 턴 actor가 명확하게 보인다.
- 내가 조작 가능한 actor가 구분된다.

---

## 8. 6단계: 토큰 선택 / 대상 정보 패널

### 목표

토큰을 클릭하면 해당 대상의 정보를 볼 수 있게 한다.

### 구현 상태

```ts
const [selectedActorId, setSelectedActorId] = useState<string | null>(null);
```

### 정보 패널에 표시할 것

```text
이름
타입
HP / Max HP
AC
상태이상
거리
현재 선택 가능한 상호작용
```

### 적 토큰 클릭 시

```text
공격 가능
사거리 부족
시야 차단
행동권 없음
대상 지정 가능
```

### 아군 토큰 클릭 시

```text
대상 지정 가능
회복 가능
버프 가능
상세 보기
```

### 완료 기준

- 토큰 클릭 시 정보 패널이 열린다.
- 빈 맵 클릭 시 선택이 해제된다.
- 대상 정보가 하단 행동 UI와 충돌하지 않는다.

---

## 9. 7단계: 좌측 파티 초상화

### 목표

파티 상태를 항상 볼 수 있게 한다.

### 표시할 것

```text
초상화
이름
HP 바
상태이상 아이콘
현재 턴 표시
조작 가능 여부
```

### 클릭 동작

```text
파티원 클릭
→ 해당 actor 선택
→ 맵에서 해당 토큰 강조
→ 정보 패널 표시
```

### 완료 기준

- 파티원 상태가 전투 중 항상 보인다.
- 초상화를 눌러 해당 캐릭터를 빠르게 선택할 수 있다.

---

## 10. 8단계: 턴 순서 바

### 목표

현재 턴과 다음 턴을 명확하게 보여준다.

### 위치

```text
전투맵 상단
```

### 표시할 것

```text
현재 라운드
현재 턴 actor
다음 턴 actor 목록
아군 / 적 구분
```

### 예시

```text
Round 2
[고블린 A] → [플레이어1] → [플레이어2] → [늑대] → [플레이어3]
```

### 완료 기준

- 현재 누구의 턴인지 즉시 알 수 있다.
- 다음 행동 순서를 예측할 수 있다.

---

## 11. 9단계: 현재 조작 가능 캐릭터 표시

### 목표

사용자가 지금 조작 가능한 캐릭터가 누구인지 명확하게 표시한다.

### 표시 기준

```text
내 캐릭터 + 내 턴: 조작 가능
내 캐릭터 + 내 턴 아님: 대기
다른 플레이어 캐릭터: 조작 불가
GM 권한: NPC/몬스터 조작 가능
관전자: 전체 조작 불가
```

### UI 표현

```text
조작 가능: 밝은 강조
조작 불가: 어둡게 처리
현재 턴 아님: 대기 표시
```

### 완료 기준

- 사용자가 자기 턴인지 혼동하지 않는다.
- 조작 불가 토큰을 눌러도 행동 버튼이 활성화되지 않는다.

---

## 12. 10단계: 행동 자원 표시

### 목표

현재 턴에 사용할 수 있는 행동 자원을 보여준다.

### 표시할 것

```text
행동: 1/1
추가 행동: 1/1
반응: 1/1
이동력: 6m / 9m
주문 슬롯: 1레벨 2/3
```

### 위치

```text
하단 행동 패널 상단
```

### 완료 기준

- 행동 버튼이 비활성화된 이유를 자원 상태와 연결해서 이해할 수 있다.
- 턴이 넘어가면 자원이 갱신된다.

---

## 13. 11단계: 하단 행동 탭

### 목표

행동을 카테고리별로 정리한다.

### 탭 구성

```text
[일반] [능력] [아이템]
```

### 일반 탭

```text
이동
기본 공격
대시
회피
도움
숨기
조사
상호작용
턴 종료
```

### 능력 탭

```text
직업 능력
종족 능력
마법
특수 행동
```

### 아이템 탭

```text
포션
스크롤
투척물
장비
퀘스트 아이템
```

### 완료 기준

- 탭 전환 시 행동 목록이 바뀐다.
- 전투 중 자주 쓰는 일반 행동이 기본 탭으로 열린다.

---

## 14. 12단계: 행동 버튼 목록

### 목표

현재 캐릭터가 사용할 수 있는 행동을 버튼으로 표시한다.

### Action 구조 예시

```ts
type CombatAction = {
  id: string;
  name: string;
  category: 'GENERAL' | 'ABILITY' | 'ITEM';
  cost: {
    action?: number;
    bonusAction?: number;
    reaction?: number;
    movement?: number;
    spellSlotLevel?: number;
  };
  range?: number;
  requiresTarget: boolean;
  targetType?: 'ENEMY' | 'ALLY' | 'SELF' | 'POINT';
  enabled: boolean;
  disabledReason?: string;
  description: string;
};
```

### 추천 API

```http
GET /api/v1/sessions/{sessionId}/combat/my-actions
```

### 완료 기준

- 현재 캐릭터 기준으로 가능한 행동 목록이 표시된다.
- 사용할 수 없는 행동도 회색 처리되어 이유를 확인할 수 있다.

---

## 15. 13단계: 행동 상세 설명

### 목표

행동을 누르거나 hover했을 때 상세 정보를 보여준다.

### 표시할 것

```text
행동 이름
소모 자원
사거리
대상 조건
효과
피해/회복/상태 변화
판정 방식
사용 불가 이유
```

### 예시

```text
장검 공격
소모: 행동 1
사거리: 근접
대상: 적 1체
명중: d20 + 근력 보정 + 숙련 보너스
피해: 1d8 + 근력 보정
```

### 완료 기준

- 사용자가 버튼을 누르기 전에 행동 결과를 대략 이해할 수 있다.
- 비활성화 행동의 이유를 확인할 수 있다.

---

## 16. 14단계: 행동 불가 사유 표시

### 목표

왜 행동이 안 되는지 명확하게 보여준다.

### 불가 사유 예시

```text
행동권이 없습니다.
추가 행동을 이미 사용했습니다.
사거리가 부족합니다.
시야가 차단되었습니다.
대상이 필요합니다.
주문 슬롯이 부족합니다.
현재 상태에서는 사용할 수 없습니다.
내 턴이 아닙니다.
조작 권한이 없습니다.
```

### UI 처리

```text
버튼 비활성화
hover/click 시 disabledReason 표시
대상 정보 패널에도 관련 사유 표시
```

### 완료 기준

- 비활성화 버튼만 보고 끝나지 않는다.
- 사용자는 다음에 뭘 해야 하는지 알 수 있다.

---

## 17. 15단계: 대상 선택 모드

### 목표

행동 선택 후 대상이나 위치를 고르게 한다.

### 상태 예시

```ts
type TargetingState = {
  actionId: string;
  targetType: 'ENEMY' | 'ALLY' | 'SELF' | 'POINT';
  selectedTargetId?: string;
  selectedPoint?: {
    x: number;
    y: number;
  };
};
```

### 안내 문구 예시

```text
공격할 대상을 선택하세요.
이동할 위치를 선택하세요.
회복할 아군을 선택하세요.
범위 중심점을 선택하세요.
```

### 완료 기준

- 행동 선택 후 화면이 대상 선택 모드로 전환된다.
- 선택 가능한 대상과 불가능한 대상이 구분된다.
- 취소 버튼으로 대상 선택 모드를 빠져나올 수 있다.

---

## 18. 16단계: 행동 확정 / 취소

### 목표

실수 클릭으로 행동이 즉시 처리되지 않게 한다.

### 추천 흐름

```text
행동 선택
→ 대상/위치 선택
→ 행동 요약 표시
→ [취소] [확정]
→ 확정 시 서버 요청
```

### 행동 요약 예시

```text
행동: 장검 공격
대상: 고블린 A
소모: 행동 1
명중: d20 + 5
피해: 1d8 + 3
```

### 추천 API

```http
POST /api/v1/sessions/{sessionId}/combat/actions
```

### 요청 예시

```ts
type ExecuteCombatActionRequest = {
  combatId: string;
  actorId: string;
  actionId: string;
  targetActorId?: string;
  targetPoint?: {
    x: number;
    y: number;
  };
};
```

### 완료 기준

- 확정 전까지 서버 상태가 바뀌지 않는다.
- 확정 후 서버가 검증하고 결과를 반환한다.
- 실패 시 실패 사유가 표시된다.

---

## 19. 17단계: 턴 종료

### 목표

현재 턴을 명확하게 넘긴다.

### 위치

```text
하단 우측 고정
```

### 추천 API

```http
POST /api/v1/sessions/{sessionId}/combat/turn/end
```

### 처리 흐름

```text
턴 종료 클릭
→ 확인 없이 바로 처리하거나 간단 확인
→ 서버에서 현재 actor의 턴 종료
→ 다음 턴 actor로 상태 갱신
```

### 완료 기준

- 턴 종료 버튼은 항상 보인다.
- 내 턴이 아닐 때는 비활성화된다.
- 턴 종료 후 턴 순서 바와 행동 자원이 갱신된다.

---

## 20. 18단계: 맵 기본 조작 보완

### 목표

전투맵을 불편하지 않게 조작할 수 있게 한다.

### 필수 조작

```text
마우스 드래그: 맵 이동
휠: 줌 인/아웃
토큰 클릭: 대상 선택
빈 칸 클릭: 선택 해제
ESC: 현재 선택/대상 선택 취소
```

### 추가하면 좋은 조작

```text
내 캐릭터로 화면 이동
현재 턴 캐릭터로 화면 이동
줌 초기화
```

### 완료 기준

- 큰 맵에서도 원하는 위치를 볼 수 있다.
- 대상 선택과 맵 이동이 충돌하지 않는다.

---

## 21. 19단계: 채팅창과 전투 화면 공존 처리

### 목표

우측 채팅창이 전투 조작을 방해하지 않게 한다.

### MVP 채팅 탭

```text
[메인] [일반]
```

### 메인 채팅

```text
GM에게 요청
NPC에게 말하기
장면 관련 대화
```

### 일반 채팅

```text
플레이어 잡담
메타 대화
```

### 주의점

- 전투 행동 처리는 채팅이 아니라 하단 행동 UI에서 처리한다.
- 채팅으로 입력한 내용이 전투 상태를 직접 변경하면 안 된다.
- AI GM은 전투 결과를 계산하지 않고 묘사만 담당해야 한다.

### 완료 기준

- 전투 중 채팅을 해도 행동 UI 상태가 꼬이지 않는다.
- 채팅과 전투 액션의 책임이 분리된다.

## 27. 핵심 기준

전투 UI는 채팅 기반으로 만들면 안 된다.

```text
대화: 채팅
행동 선택: 하단 UI
대상 선택: 맵 / 토큰
상태 변경: 백엔드 검증
결과 묘사: GM 또는 AI
```

이 기준만 지키면 MVP 이후 기능을 추가해도 구조가 크게 무너지지 않는다.
