# Content and License Rules

## 1. 제품 데이터에는 권리가 확인된 콘텐츠만 넣는다

룰, 시나리오, 이미지, 지도, 몬스터, NPC, 텍스트는 라이선스가 명확해야 한다.

허용:

- D&D 5e SRD에서 공개된 규칙, 클래스, 주문, 몬스터, 장비, 조건, 판정 구조
- 명시적인 무료 라이선스가 있는 시나리오
- CC-BY-4.0 또는 호환 가능한 라이선스 콘텐츠
- 팀이 직접 작성한 오리지널 시나리오, NPC, 지역, 몬스터 변형
- 사용자가 직접 입력한 세션 내 플레이 로그와 캐릭터 정보

금지:

- SRD에 포함되지 않은 유료 룰북 전문 또는 고유 설정
- 별도 IP에 속한 세계관, 지명, 캐릭터, 고유 몬스터
- 라이선스가 불명확한 팬 시나리오, 이미지, 지도, 텍스트
- 무료 배포 PDF라도 재배포나 제품 포함 권리가 명확하지 않은 원문 텍스트

이유:

TRPG 콘텐츠는 룰, 설정, 이미지 권리가 복잡하다. 개발 참고 자료로 볼 수 있는 것과 제품 데이터로 포함할 수 있는 것은 다르다.

## 2. 로컬 참고 자료를 저장소에 넣지 않는다

로컬 PDF나 외부 자료는 개발 참고용으로만 사용한다.

지켜야 할 것:

- 로컬 룰북 PDF를 저장소에 커밋하지 않는다.
- seed 데이터에는 권리가 확인된 내용만 넣는다.
- SRD 원문을 대량 복제하지 않는다.
- 엔진에 필요한 구조화 데이터와 짧은 근거 참조 중심으로 사용한다.

이유:

저장소에 들어간 자료는 배포와 재사용 가능성이 생긴다. 권리 범위가 불명확한 자료는 로컬 참고와 제품 포함을 명확히 분리해야 한다.
