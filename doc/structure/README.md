# Structure Documents

이 폴더는 프로젝트의 현재 구조를 설명하는 기준 문서를 모아둔다.

`rules/`가 개발과 설계에서 지켜야 할 원칙을 설명한다면, `structure/`는 현재 시스템이 어떤 구성요소와 흐름으로 동작하는지 설명한다. 새 기능을 구현할 때는 관련 구조 문서와 원칙 문서를 함께 확인한다.

## 공통 문서 구조

모든 `structure/` 문서는 가능한 한 아래 흐름을 따른다.

```md
# 문서 제목

## 문서 목적

## 적용 범위

## 핵심 요약

## 상세 내용

## 관련 원칙

## 관련 문서

## 변경 시 주의사항
```

본문의 세부 구조는 문서 유형에 맞게 조정한다. 정보 손실을 막기 위해 모든 문서에 같은 하위 제목을 강제하지 않는다.

## 문서 유형

| 유형 | 대상 문서 | 본문 패턴 |
| --- | --- | --- |
| 개요형 | `PRODUCT_OVERVIEW.md` | 핵심 방향, 주요 구조 문서, 참고 문서 |
| 범위/기준형 | `PRODUCT_SCOPE.md`, `QUALITY_MVP_ACCEPTANCE.md`, `RULESET_SRD5E_MVP.md` | 포함 범위, 제외 범위, 기준표, 완료/판정 조건 |
| 흐름형 | `RUNTIME_SESSION_TURN_FLOW.md`, `trpg_main_command_mvp_flow_with_categories.md` | 입력, 처리 단계, 출력, 실패/예외, 검증 기준 |
| 계약형 | `AI_RUNTIME_CONTRACTS.md` | 공통 조건, 역할별 입력, 역할별 출력, 검증/실패 처리 |
| 구조 참조형 | `ERD_MVP_SESSION_SERVICE_MODEL.md`, `SCREEN.md` | 구성요소 목록, 요소별 책임, 관계/전이, 주의사항 |

## 작성 원칙

- 구조 설명과 설계 원칙을 섞어 길게 반복하지 않는다.
- 여러 문서에 반복 적용되는 원칙은 `../rules/`에 둔다.
- 현재 구현 기준과 과거 초안이 섞이면 현재 구현 기준을 먼저 배치한다.
- 예시는 `../examples/`에 두고 구조 문서에서는 필요한 만큼만 링크한다.
- 시나리오 본문은 `../scenarios/`에 둔다.
- 완료된 계획이나 과거 초안은 `../completed/`에 둔다.
