# 인간 GM 세션 플레이 화면 및 전투 로직 통일 작업 정리

작성일: 2026-05-31
대상 작업 기간: 2026-05-30 ~ 2026-05-31

## 요약

인간 GM 세션 플레이 화면의 탐색/전투 노드 사용성을 보강하고, 전투 노드에서 인간 GM이 몬스터를 플레이어 캐릭터와 비슷한 조작감으로 다룰 수 있도록 전투 액터 로직을 점진적으로 통일하는 작업을 진행했다.

큰 방향은 다음과 같다.

- 탐색 노드의 하단 UI와 GM 전용 패널을 실제 플레이 화면에 맞게 정리한다.
- 시야/벽/탐험 기억/이동 처리의 게임 경험과 성능 병목을 개선한다.
- 전투 노드에서 인간 GM은 플레이어 시야에 막히지 않고 몬스터를 조작할 수 있게 한다.
- 플레이어 캐릭터와 몬스터의 턴 조작 로직을 "현재 액터 + 권한" 모델로 합쳐가되, AI GM의 자동 몬스터 조작 흐름은 유지한다.

2026-05-31 현재, 탐색 노드 UI와 시야/이동 관련 주요 개선은 1차 적용되었고, 전투 노드에서는 인간 GM이 현재 몬스터 턴에 몬스터 토큰 이동, 공격, 대시, 회피, 숨기까지 같은 하단 행동 UI 흐름에서 조작할 수 있는 단계까지 진행했다.

## 2026-05-30 ~ 2026-05-31 주요 작업

### 탐색 노드 하단 UI

- 선택 대상이 플레이어 캐릭터 토큰일 때만 선택 캐릭터 카드와 인벤토리 카드를 표시하도록 정리했다.
- 오브젝트, 타일, 빈 공간 등 토큰이 아닌 대상을 선택했거나 선택이 없을 때는 중앙 선택 대상 행동 카드가 양옆 공간을 대신 차지하도록 변경했다.
- 탐색 노드의 선택 캐릭터 카드는 전투 노드의 캐릭터 카드와 비슷하게 HP, 이동, AC, 상태, 인벤토리 요약 등 존재 이유가 보이는 정보 중심 카드로 보강했다.
- 하단 UI 중앙의 선택 대상 행동 버튼들이 작은 화면이나 확대 상태에서 여러 줄로 늘어나 맵을 과도하게 가리는 문제를 줄이도록 버튼 레일/스크롤 구조와 중앙 정렬을 조정했다.

### 탐색 노드 시야, 벽, 탐험 기억

- 일반적인 타일 기반 턴제 게임의 시야 경험을 기준으로, 현재 시야와 벽 시스템을 재검토했다.
- 벽/차단 오브젝트를 기준으로 시야가 막히고, 플레이어가 본 영역은 탐험 기억으로 남는 방식으로 개선했다.
- 탐험 기억은 새로고침 후에도 유지되도록 저장 경로를 보강했다.
- 탐험 기억은 "현재 보이는 영역"이 아니라 "이전에 본 적 있는 영역"을 어둡게 남겨 플레이어가 탐색한 지형을 기억하게 하는 기능이다.
- 토큰 이동 시 버벅거림을 유발하던 주요 원인을 점검하고, 이동 중 반복 계산과 렌더링 부담을 줄이는 방향으로 일부 최적화를 적용했다.

### 탐색 노드 맵 경계 및 접이식 패널

- 맵 가장자리에 벽이 없을 때 이동 가능한 타일을 따라 맵 바깥으로 계속 나갈 수 있던 버그를 수정했다.
- GM 전용 우측 패널을 오른쪽으로 접고 펼칠 수 있게 만들었다.
- 패널 상단 텍스트 토글 대신 패널 왼쪽 중앙의 화살표 버튼으로 접기/펼치기 조작을 바꿨다.
- 채팅창도 같은 방식으로 접고 펼칠 수 있게 했다.
- GM 패널/채팅창 접기 버튼의 과한 튕김 애니메이션은 제거했다.

### 전투 노드 인간 GM 시야

- 인간 GM 전투 화면에서는 플레이어 시야 밖 영역을 완전히 가리지 않고 살짝 어둡게만 표시하도록 변경했다.
- 이를 통해 인간 GM은 전체 전투 맵을 보고 플레이어 시야 밖의 몬스터도 조작할 수 있다.
- 플레이어 화면의 시야 제한 경험은 유지하고, GM 화면에만 "관리자용 전체 시야 + 플레이어 시야 프리뷰"에 가까운 표현을 적용했다.

### 전투 노드 인간 GM 몬스터 조작

- 전투 맵에서 현재 턴 몬스터 토큰을 인간 GM이 직접 조작할 수 있게 했다.
- 몬스터 참여자가 `tokenId`를 갖지 않는 경우에도, 이름이 유일하게 일치하는 적대 토큰을 fallback으로 찾아 조작할 수 있게 보강했다.
- 전투 하단 행동 자원 패널이 GM 화면에서는 내 플레이어 캐릭터가 아니라 현재 액터 몬스터의 HP, 이동, 행동/추가 행동/반응 자원을 표시하도록 바꿨다.
- 인간 GM이 현재 몬스터 턴일 때 공격 버튼으로 대상 플레이어 캐릭터를 선택하고 공격할 수 있게 했다.
- 이후 같은 공통 액터 행동 경로를 통해 몬스터의 대시, 회피, 숨기도 사용할 수 있게 확장했다.

## 완료한 작업

### 탐색 노드

- 선택 대상이 플레이어 캐릭터 토큰일 때만 캐릭터/인벤토리 카드를 표시하고, 그 외 선택 상태에서는 중앙 행동 카드가 확장되도록 수정했다.
- 탐색 노드 캐릭터 카드를 전투 노드 카드와 비슷한 정보 밀도로 보강했다.
- 하단 행동 버튼이 작은 화면에서 여러 줄로 늘어나 맵을 가리는 문제를 줄이도록 버튼 레일과 중앙 정렬을 조정했다.
- 시야/벽/탐험 기억 모델을 1차 개선하고, 탐험 기억이 새로고침 후에도 유지되도록 저장 경로를 보강했다.
- 토큰 이동 버벅임 원인을 점검하고, 이동 중 반복 계산과 렌더링 부담을 일부 줄였다.
- 맵 경계 밖으로 이동 가능한 버그를 수정했다.
- GM 우측 패널과 채팅창 접기/펼치기를 추가하고, 접기 버튼의 과한 튕김 애니메이션을 제거했다.

### 전투 노드

- 인간 GM 화면에서는 플레이어 시야 밖도 살짝 어둡게만 표시해 전체 맵과 몬스터를 조작할 수 있게 했다.
- 현재 액터 기반 전투 UI를 도입해 GM 몬스터 턴에는 현재 몬스터의 HP, 이동, 행동/추가 행동/반응 자원을 표시하도록 했다.
- 인간 GM이 현재 몬스터 턴에 몬스터 이동, 공격, 대시, 회피, 숨기를 같은 하단 행동 UI 흐름으로 조작할 수 있게 했다.
- AI GM 자동 몬스터 턴과 HUMAN GM 수동 몬스터 공격이 같은 공용 몬스터 공격 helper를 사용하도록 정리했다.
- `combat/actor/action` 공통 액터 endpoint를 추가하고, 몬스터 공격과 기본 행동을 연결했다.
- 직접 공격 endpoint로 현재 턴이 아닌 몬스터가 공격하는 우회 경로를 막았다.
- 공격 action/reaction/bonus action 비용 소모를 공격 굴림보다 앞쪽으로 이동해, 자원이 없을 때 명중/피해 주사위와 로그가 발생하지 않도록 했다.
- Shield 반응 대기 분기에서 공격 action이 한 번만 소모되고 피해/로그/전투 업데이트는 반응 처리 전까지 발생하지 않도록 회귀 테스트로 고정했다.
- 공통 액터 endpoint로 실행한 인간 GM 몬스터 공격도 Shield 반응 대기 분기에서 같은 비용 선소모/피해 지연 흐름을 타도록 회귀 테스트로 고정했다.
- 몬스터 행동 후보(`monsterActions`)를 전투 응답에 포함하고, 카탈로그/SRD/fallback 후보를 중복 제거해 노출하도록 했다.
- 몬스터 행동 후보 응답에 비용 타입(`costType`)을 추가해, 이후 bonus/reaction/limited-use 몬스터 능력 후보를 같은 모델로 확장할 기반을 마련했다.
- 몬스터 원거리 행동의 일반/장거리 사거리 표시와 장거리 불리점 처리를 추가했다.
- HUMAN GM 세션에서 몬스터 턴 종료 후 다음 몬스터가 와도 AI 자동 몬스터 루프가 이어서 실행되지 않도록 회귀 테스트로 고정했다.

## 전투 턴 로직 통일 계획

현재 전투 로직은 과거 구조상 플레이어 캐릭터와 몬스터가 다른 길을 탔다.

- 플레이어 캐릭터는 UI 버튼 -> 전용 endpoint -> `CombatService`의 플레이어 중심 행동 메서드로 이어진다.
- AI GM 몬스터는 서버 자동 턴 루프 -> `autoMonsterTurn` -> 몬스터 행동 선택/이동/공격으로 이어진다.
- 인간 GM 몬스터 조작은 기존에 명확한 수동 조작 경로가 부족했다.

통일 방향은 "행동 규칙을 하나로 만들고, 조작 가능 여부만 권한으로 갈라내는 것"이다.

### 1. 현재 액터 모델 정리

- 전투의 기준 조작 대상은 항상 `combat.currentParticipantId`가 가리키는 현재 액터로 본다.
- 플레이어 화면에서는 현재 액터가 자기 세션 캐릭터일 때만 조작 가능하다.
- 인간 GM 화면에서는 현재 액터가 적대 몬스터일 때 조작 가능하다.
- 호스트 권한은 필요한 GM override와 수동 몬스터 조작에만 사용한다.

진행상황: 1차 적용 완료.

- 프론트 전투 노드에 `activeCombatActor`, `activeActionResources`, `canControlActiveActor` 개념을 도입했다.
- 전투 맵의 이동 가능 토큰도 현재 조작 가능한 액터 토큰 기준으로 제한했다.

### 2. 토큰 이동 로직 통일

- 플레이어 캐릭터 이동과 인간 GM 몬스터 이동이 같은 전투 이동 endpoint와 서버 검증을 타도록 한다.
- 이동 가능 범위, 기회공격, 지형 효과, 이동력 차감은 actor type이 아니라 현재 액터와 권한에 의해 결정한다.

진행상황: 1차 적용 완료.

- 인간 GM도 현재 몬스터 턴에는 몬스터 토큰을 전투 이동 경로로 이동시킬 수 있다.
- 토큰 lookup fallback을 보강해 tokenId가 비어 있는 몬스터 참여자도 유일한 적대 토큰 이름으로 찾을 수 있다.

### 3. 공격 행동 로직 통일

- 플레이어 장착 무기 공격과 몬스터 기본 공격은 수치 산출 방식은 다르더라도 최종 명중/피해/반응/로그 처리는 가능한 한 같은 `resolveAttack` 계열을 사용한다.
- AI GM 자동 몬스터 턴과 인간 GM 수동 몬스터 공격이 같은 몬스터 공격 처리 helper를 공유한다.

진행상황: 1차 적용 완료.

- `CombatActorActionDto`와 `POST /sessions/:sessionId/combat/actor/action` 경로를 추가했다.
- 인간 GM의 몬스터 공격 버튼은 `resolveCombatActorAction`을 통해 새 공통 액터 행동 endpoint를 호출한다.
- 서버에서는 몬스터 행동 선택, 사거리 체크, 공격 처리, 자동 턴 종료 처리를 공용 helper로 분리했다.
- 기존 `autoMonsterTurn`도 같은 공용 몬스터 공격 helper를 사용하도록 정리했다.

### 4. 기본 행동 로직 통일

- 대시, 회피, 숨기 같은 기본 행동은 플레이어 캐릭터와 몬스터가 같은 내부 helper를 사용하게 한다.
- UI는 현재 액터의 행동 자원을 기준으로 버튼 활성화를 결정한다.

진행상황: 1차 적용 완료.

- 서버의 기존 플레이어 전용 `dash`, `dodge`, `hide` 구현을 내부 공용 액터 행동 helper로 분리했다.
- 새 `combat/actor/action` 경로에서 `actionType: dash | dodge | hide`를 처리하도록 확장했다.
- 전투 UI의 일반 행동 버튼이 GM 몬스터 턴에는 현재 몬스터의 행동 자원을 기준으로 활성화되고, 클릭 시 새 액터 행동 endpoint를 호출한다.

### 5. 남은 통일 작업

아직 완전히 통일되지 않은 부분은 다음과 같다.

- 플레이어의 장착 무기 공격, 보조 공격, 암습, 주문 시전은 아직 각각의 전용 endpoint를 사용한다.
- 몬스터 행동 후보 UI는 1차 확장되어 catalog/SRD/fallback 기반 행동 후보를 버튼으로 보여주지만, multiattack/recharge/save-based action 전체 실행 모델은 아직 없다.
- 몬스터 주문, 몬스터 특수 능력, condition rider, limited-use ability는 더 넓은 실행 모델이 필요하다.
- AI GM 자동 몬스터 행동 선택은 유지되지만, HUMAN GM이 같은 후보 목록을 보고 직접 선택하는 UI는 아직 1차 MVP 수준이다.
- `combat/actor/action`은 현재 기본 행동과 몬스터 공격 중심으로 확장되었고, 모든 플레이어 행동까지 완전히 흡수한 상태는 아니다.

참고:

- 공통 액터 행동 경로에 대해 인간 GM이 현재 몬스터에게 대시/회피/숨기/선택 actionId 공격을 실행하는 회귀 테스트를 추가했다.
- 비 GM 사용자가 공통 액터 행동 경로로 몬스터를 조작하려 할 때 `GM_403`으로 거부되는 회귀 테스트를 추가했다.
- HUMAN GM 세션에서도 직접 공격 endpoint로 현재 턴이 아닌 몬스터가 공격하지 못하도록 막고, 피해/로그/이벤트가 발생하지 않는 회귀 테스트를 추가했다.
- 전투 응답의 `monsterActions` 후보가 몬스터 참가자에게만 내려가고, 카탈로그/룰 엔진 중복 actionId를 제거하며 fallback 행동을 함께 노출하는 회귀 테스트를 추가했다.
- `monsterActions` 응답에 장거리 사거리(`longRangeFt`)와 후보 신뢰도(`confidence`)를 추가하고, 전투 UI 툴팁에서 원거리 몬스터 행동을 `80/320ft`처럼 표시할 수 있게 했다.
- `monsterActions` 응답에 비용 타입(`costType`)을 추가하고, 카탈로그 후보는 실제 비용 타입을, SRD/fallback 후보는 기본 `action` 비용으로 내려가도록 했다.
- 원거리 몬스터 행동은 정상 사거리 밖이지만 장거리 사거리 안이면 거부하지 않고 불리점(`DISADVANTAGE`)으로 공격 굴림을 처리하도록 수정했다.
- 몬스터 숨기 실패, 사거리 밖 공격 거부, 공격 후 `autoEndTurn` 턴 전환, HUMAN GM 세션에서 다음 몬스터 턴을 AI가 자동 실행하지 않는 회귀 테스트를 추가했다.
- 기본 행동의 action 자원이 이미 소진된 경우 공통 액터 행동이 중단되고 후속 효과/로그/이벤트가 발생하지 않는 회귀 테스트를 추가했다.
- 공격 행동의 action 자원이 이미 소진된 경우 피해 적용, 턴 로그, 전투 업데이트 이벤트가 발생하지 않는 회귀 테스트를 추가했다.
- Shield 반응 대기 분기에서 공격 action이 한 번만 소모되고 피해/로그/전투 업데이트는 반응 처리 전까지 발생하지 않는 회귀 테스트를 추가했다.
- 공통 액터 endpoint의 인간 GM 몬스터 공격에서도 Shield 반응 대기 시 공격 action이 한 번만 소모되고 피해/로그/전투 업데이트가 발생하지 않는 회귀 테스트를 추가했다.
- 공격 경로의 action/reaction/bonus action 비용 소모를 공격 굴림보다 앞쪽으로 이동해, 자원이 없을 때는 명중/피해 주사위도 굴리지 않도록 수정했다.
- 이 과정에서 전투 이동 blocker 판정이 지형 셀까지 막고 있어 어려운 지형/위험 지형 진입이 막히는 런타임 버그를 확인하고 수정했다.
